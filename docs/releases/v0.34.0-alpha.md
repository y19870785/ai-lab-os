# AI-Lab OS v0.34.0 Alpha

**成熟度：Alpha / local-first / single-user-oriented**  
**状态：Release Authorized; GitHub publication pending final release operation**

本文件定义 v0.34.0 相对 v0.33.0 的 Alpha 发布范围。它不是生产可用性声明；SP-015、SP-015A 与 SP-015R 均已合并、验证并封存，Owner 与 ChatGPT 已授权 `v0.34.0` GitHub Pre-release。GitHub publication pending final release operation，实际 Tag、Release URL 与发布时间以 GitHub 为权威来源。

## 用户可见能力

- Canonical UserTask：真实持久化、查询、状态转换与 CEO Assistant 接入。
- Reminder：确定性今天/明天时间子集、持久化调度、站内状态、列表、详情、取消和改期。
- Intent Safety：只读查询、显式写命令与普通聊天的稳定 effect 边界。
- Daily Agenda：统一读取 UserTask、Reminder 与 Work Log。
- Unified Inbox / Capture-to-Action：捕获未分类事项，并显式转化为 UserTask、Reminder、Work Log、Note 或 Dismiss。
- 中文 Reminder 小时：在明确 `上午/下午/晚上` 时支持 `一` 至 `十二`，保留既有半、一刻与数字分钟组合。
- API、CLI 与 CEO Assistant：共享 Composition Root、Workspace 边界和正式领域服务。

## 主要架构变化

- `core.system.create_system()` 成为 API、CLI、CEO Assistant 与集成测试的 canonical Composition Root。
- `FailureInfo` 统一跨 Runtime、领域服务和 API 的失败语义。
- DatabaseManager 收敛受管 Memory SQLite 连接所有权。
- UserTask、Reminder、Scheduler、Daily Agenda 与 Unified Inbox 使用正式服务和持久化状态，不以 LLM 输出或日志作为用户可见真相。
- `inbox_resolution_claims` 在任何目标写入前确定唯一解析类型，为跨进程竞争与崩溃恢复提供正确性边界。
- 根 `project_state.json` 成为唯一机器可读仓库治理状态与稳定发布授权源；`pyproject.toml` 继续是唯一运行时产品版本源，当前 Git/GitHub 对象按需查询。

## 数据兼容

- 本候选版本不修改数据库 Schema。
- 既有 UserTask、Reminder、Scheduler、Memory、Agenda 与 Inbox 数据路径保持不变。
- 不删除历史迁移或兼容层。
- 跨 SQLite 业务继续使用 Saga、幂等和 reconciliation，不宣称跨库原子事务。

升级前仍建议备份本地数据目录；当前项目没有正式的企业级 backup/restore 产品能力。

## 默认关闭或需显式配置

- Reminder / Scheduler：实现已集成，启动状态由配置控制。
- Knowledge：基础实现存在，默认关闭；真实主链路未完成。
- Coordination：基础实现存在，默认关闭且未接入 CEO Assistant 主链路。
- Real Provider：需要安装对应 extra、配置凭据并获得单独授权；普通 Quality Gate 不调用外部模型。

## 安装与升级

要求 Python 3.11 或更高版本。

```powershell
python -m pip install -e ".[local]"
python -m pytest tests/governance -q
python -m pytest tests --ignore=tests/real -m "not real" -q --tb=no
python -m build
```

运行时版本由 `pyproject.toml` 派生。升级后可以使用以下命令确认：

```powershell
python -c "import core; print(core.__version__)"
```

预期 Alpha 版本为 `0.34.0`。
