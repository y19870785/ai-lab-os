# CEO Assistant MVP 需求文档

## 版本

v0.32.0 — Alpha Product

## 产品定位

CEO Assistant 是超哥的个人工作总控助手，运行在 AI-Lab 平台上。

第一版不追求功能数量，追求：能真实运行、每天可用、数据可持续积累。

## 核心功能

### 1. 工作记录 (Work Log)

用户输入自然语言描述工作事项，系统自动：
- 保存原始记录
- 提取日期、对象、事项、状态、标签
- 写入 Episodic Memory
- 必要时生成 Semantic Memory
- 返回结构化确认结果

示例：
```
输入: "今天和张经理确认了蜂蜡面包袋检测方案，等待客户回复。"
输出:
  日期: 2026-07-13
  对象: 张经理
  事项: 蜂蜡面包袋检测方案确认
  状态: 等待客户回复
  标签: [蜂蜡, 食品检测, 客户沟通]
```

### 2. 待办任务 (Task)

用户输入自然语言创建任务，系统自动：
- 创建 Task 记录
- 保存截止时间、优先级、状态
- 关联原始工作记录
- 支持查询、暂停、完成、取消

示例：
```
输入: "提醒我明天下午跟进张经理的 FDA 检测结果。"
输出:
  任务: 跟进张经理 FDA 检测结果
  截止: 2026-07-14 14:00
  优先级: 高
  状态: 待办
```

### 3. 决策记录 (Decision)

用户记录重要决策，系统自动：
- 创建 Decision Memory
- 保存 trigger、alternatives、chosen、reason
- 关联相关任务和工作记录
- 后续可追踪结果

示例：
```
输入: "这次蜂蜡面包袋检测先按 21 CFR 175.300 做，不先做完整迁移测试。"
输出:
  决策: 优先 21 CFR 175.300 检测
  理由: [用户提供的理由]
  关联任务: [蜂蜡面包袋检测方案]
```

### 4. 知识问答 (Knowledge QA)

用户导入 Markdown/TXT 文档后，可以基于文档提问：
- 检索当前 Workspace Knowledge
- 返回引用来源
- 标注检索置信度
- 无可靠来源时明确说明

### 5. 每日简报 (Daily Brief)

生成基于真实数据的每日简报：
- 今日待办
- 已逾期任务
- 最近新增工作记录
- 待跟进事项
- 最近决策
- 当前风险或阻塞
- 建议优先处理的三项工作

要求：所有内容来自真实数据，LLM 只做组织和表达。

### 6. 多轮对话 (Multi-turn)

在同一个 Session 中保持上下文连贯，支持连续工作。

## 非功能需求

- 支持 Mock 和 DeepSeek 两种模式
- 重启后数据可恢复
- Workspace 数据隔离
- 不记录 API Key
- 不暴露本地绝对路径
- 不保存模型私有思维链
- 知识来源与用户输入明确分离

## 接入方式

- CLI: `python -m cli brief / log / task / decide / ask / chat`
- REST API: `GET/POST /brief`, `/work-logs`, `/tasks`, `/decisions`, `/knowledge`

## 验收标准

- 6 个核心闭环全部通过
- Mock 和 DeepSeek 模式均可用
- 647 历史测试零回归
- 安全审查通过
